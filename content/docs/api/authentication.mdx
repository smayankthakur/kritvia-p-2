# Authentication

The Kritvia API uses API keys to authenticate requests. This guide explains how to manage and use your API keys.

## API Keys

Your API keys are used to authenticate requests to the Kritvia API. You can create and manage API keys in the [Kritvia Dashboard](https://dashboard.kritvia.ai).

### Creating an API Key

1. Log in to the [Kritvia Dashboard](https://dashboard.kritvia.ai)
2. Navigate to **Settings** → **API Keys**
3. Click **Create API Key**
4. Give your key a name and select permissions
5. Copy the key (it will only be shown once)

## Using Your API Key

Include your API key in the `Authorization` header:

```bash
curl -X POST https://api.kritvia.ai/v1/generate \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Hello"}'
```

Or in JavaScript:

```javascript
const response = await fetch("https://api.kritvia.ai/v1/generate", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    prompt: "Hello"
  })
})
```

## SDK Authentication

When using the SDK, pass your API key during initialization:

```javascript
import { Kritvia } from "@kritvia/sdk"

const kritvia = new Kritvia({
  apiKey: process.env.KRITVIA_API_KEY
})
```

## Environment Variables

We recommend storing your API key in environment variables:

```bash
# .env
KRITVIA_API_KEY=your_api_key_here
```

## Key Permissions

When creating an API key, you can set the following permissions:

| Permission | Description |
|------------|-------------|
| `read` | Read access to projects and data |
| `write` | Write access to create and update resources |
| `delete` | Delete access to remove resources |
| `admin` | Full administrative access |

## Security Best Practices

1. **Never commit API keys** to version control
2. **Use environment variables** to store keys
3. **Rotate keys regularly** - create new keys before revoking old ones
4. **Use minimal permissions** - only grant the permissions you need
5. **Monitor usage** - check your dashboard for unusual activity

## Revoking API Keys

If you suspect your API key has been compromised:

1. Go to **Settings** → **API Keys** in the dashboard
2. Find the compromised key
3. Click **Revoke**
4. Create a new key

## Rate Limits

API keys are subject to rate limits:

| Plan | Requests/minute |
|------|-----------------|
| Free | 60 |
| Pro | 600 |
| Enterprise | Custom |

If you exceed your rate limit, you'll receive a `429 Too Many Requests` response with a `Retry-After` header.

## Error Responses

### 401 Unauthorized

```json
{
  "error": {
    "code": "INVALID_API_KEY",
    "message": "The provided API key is invalid"
  }
}
```

### 403 Forbidden

```json
{
  "error": {
    "code": "INSUFFICIENT_PERMISSIONS",
    "message": "Your API key does not have the required permissions"
  }
}
```
